Section I: Bezier Curves and Surfaces

Part 1: Bezier Curves with 1D de Casteljau Subdivision

Briefly explain de Casteljau's algorithm and how you implemented it in order to evaluate Bezier curves.

de Casteljau's algorithm consists of recursively dividing lines into segments by a proportion t (so that one segment has length l*t and the other has length l*(1-t)), then drawing a new line through the divisions, until there is one line left. The corresponding division point of that line is a spot on the bezier curve.
To implement this, we store the list of vertices, construct line segments, and then perform the recursive division and construction as described above. As t changes from 0 to 1, the final point drawn traces the shape of the bezier curve, so we can draw the curve by drawing all points corresponding to 0 < t < 1.

Part 2: Bezier Surfaces with Separable 1D de Casteljau

Briefly explain how de Casteljau algorithm extends to Bezier surfaces and how you implemented it in order to evaluate Bezier surfaces.

To extend de Casteljau's algorithm to a surface, we first use de Casteljau's algorithm with parameter u to get a set of bezier curves, which are defined by their control points. We then re-run the algorithm on that set of points with parameter v, which gives a point on the bezier surface. Since our point is parameterized by both u and v, we know that this point corresponds to the point (u, v) on the bezier surface.

Part 6: Loop Subdivision for Mesh Upsampling

Briefly explain how you implemented the loop subdivision and describe any interesting implementation / debugging tricks you have used.

To implement loop subdivision, we first computed the new positions for all vertices in the original mesh, where the new position was a sort of weighted average of its adjacent vertices and its original position. We then calculated the position of any vertices that would be created, using another weighted average of the vertices around the edge that would be split. We then iterated through every edge in the original mesh and split it using the function from part 5. Because this splitting also created new edges that would be added to the list to iterate through, we made sure to mark those edges as new and to not split them further, preventing an infinite loop. After splitting an edge we store the position calculated in the previous step inside the vertex (but not actually updating the position of the vertex yet). We then flipped any new edge that connected an old and a new vertex. Finally, we updated all vertex positions with the precomputed and stored values.
While debugging, we found that while it didn't seem like which halfedge a newly created vertex pointed to would matter, it did cause our implementation to be off as the adjacent vertex that we thought was close by was actually further away. Additionally, some of the hafledges were originally accessed using edge->hafledge->vertex->halfedge (which was thought to be equivalent to edge->halfedge), but this resulted in giving a different halfedge than expected because the halfedge tied to a vertex is basically random. To debug problems we used the edge inspector and recorded the addresses of specific mesh elements, then found them again after doing the loop subdivision process and checked if they were the expected values.
